# TruPath — System & Company Overview

## 1. Company Definition

**TruPath** is a coaching and transformation company built on a proprietary behavioral framework that identifies, predicts, and reshapes human behavior across multiple life domains.

### Delivery Model
TruPath primarily delivers its work through:
- 1:1 coaching
- Small group coaching
- Mid-size group programs
- Live in-person workshops
- Live virtual courses

The coaching experience is supported and enhanced by a software system that enables scalable assessment, analysis, and personalized outputs.

---

## 2. Core Intellectual Property (IP)

At the center of TruPath is a **behavioral pattern engine**.

This engine:
- Diagnoses underlying behavioral patterns
- Explains current outcomes
- Predicts future behavior and environment
- Provides targeted interventions

---

## 3. The Behavioral Pattern Engine

### 3.1 Foundational Structure

The system is built on:

#### Three Domains of Disconnection
1. **Disconnection from Self** (low self-worth)
2. **Disconnection from Others** (conditional love)
3. **Disconnection from All That’s Greater** (lack of trust / fear / control)

#### Active vs Passive Expression
Each domain contains:
- One **active** pattern (doing/creating)
- One **passive** pattern (allowing/withdrawing)

---

### 3.2 The Six Core Patterns

| Domain | Active Pattern | Passive Pattern |
|--------|--------------|----------------|
| Self | False Self-View | External Validation |
| Others | Showing Love | Receiving Love |
| Greater | Control | Fear |

---

### 3.3 Pattern Structure

Each pattern is defined across four components:

1. **Core Belief / Untruth**
2. **Observable Behaviors**
3. **Emotional States**
4. **Measurable Consequences**

---

### 3.4 Reinforcing Loop Mechanism

Each pattern operates as a closed loop:

Belief → Behavior → Outcome → Reinforces Belief

These loops:
- Are self-sustaining
- Produce predictable outcomes
- Can be interrupted through targeted intervention

---

## 4. Multi-Pattern System

Individuals typically exhibit:
- One **primary pattern**
- One or more **secondary patterns**

The system includes:
- Co-occurrence modeling
- Reinforcement relationships between patterns
- Pattern interaction logic

This allows for:
- More accurate diagnosis
- Deeper behavioral insight
- Better intervention targeting

---

## 5. Domain Expansion Model

TruPath applies the same pattern engine across four major life domains:

### Core Domains
1. **Finance**
2. **Relationships** (personal and professional)
3. **Health**
4. **Purpose** (career, job, calling)

---

### 5.1 Key Principle

The pattern is primary. The domain is the expression.

The same underlying pattern:
- Appears in all domains
- Drives consistent behavior
- Produces similar outcomes in different contexts

---

## 6. Entry Point Model (“4 Doors”)

Each domain acts as an independent entry point into the system.

### Process Flow

1. A user enters through a domain where they feel pain (e.g., finance)
2. The system identifies their underlying pattern(s)
3. The same pattern is mapped across:
   - Relationships
   - Health
   - Purpose
4. The user gains:
   - Domain-specific insight
   - Cross-domain awareness

---

## 7. Predictive Behavioral Capability

The system enables prediction of:

- Financial behaviors (spending, saving, debt patterns)
- Decision-making tendencies
- Risk behaviors
- Relationship dynamics
- Environmental outcomes

### Core Logic

Pattern → Behavior → Environment

Because patterns consistently drive behavior, and behavior shapes outcomes, the system can anticipate results with high reliability.

---

## 8. Intervention System

Once patterns are identified, TruPath applies:

- Pattern-specific interventions
- Domain-aware actions
- Behavioral and financial adjustments

These interventions are designed to:
- Interrupt reinforcing loops
- Replace underlying beliefs
- Create new behavioral outcomes

---

## 9. Software System (Support Layer)

TruPath includes a software suite that supports delivery and scale.

### Capabilities

- Captures user input (forms, assessments)
- Processes responses
- Identifies patterns
- Generates insights
- Produces personalized outputs (reports, guidance)

### Purpose

- Scale coaching insights
- Standardize analysis
